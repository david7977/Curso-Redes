Capítulo 12 – Estructura y componentes OSI

🧾 Explicación General

El modelo OSI divide la comunicación en 7 capas, cada una con funciones específicas: desde la transmisión física (cables, señales) hasta la interacción del usuario (navegador, correo, videollamadas). Fue creado por la ISO para estandarizar la interoperabilidad entre fabricantes y sigue siendo clave para diagnosticar fallos, diseñar redes y entender cómo viajan los datos.

🧩 Conceptos Clave (como si tuvieras 10 años)

  • Imagina un tren 🚆 con 7 estaciones: cada estación hace algo distinto para que el viaje de los datos sea seguro.
  • Si una estación falla (ejemplo: la de control de billetes = capa de enlace), el tren se atasca.
  • Las últimas estaciones son las que tú usas: abrir el navegador, mandar un mail o hacer una videollamada.
  • Cada capa tiene su “tarea”: unas mueven los vagones, otras organizan, otras traducen, y la última te entrega el paquete.

📚 Desglose

  • Física: señales eléctricas, ópticas o inalámbricas, cables, conectores.
  • Enlace de datos: controla errores, divide en tramas, subcapas MAC y LLC.
  • Red: enrutamiento, direcciones IP, fragmentación, congestión.
  • Transporte: segmentos, orden correcto, control de flujo, TCP.
  • Sesión: apertura, mantenimiento y cierre de sesiones entre aplicaciones.
  • Presentación: traduce, comprime, cifra/descifra datos.
  • Aplicación: la que usas directamente: HTTP, FTP, SMTP.

🎭 Historia / Analogía

La comunicación es como un tren 🚆. Las vías son la capa física, el controlador es la capa de enlace, el GPS es la capa de red, la logística que reparte los vagones es la capa de transporte, las estaciones abiertas son la capa de sesión, el traductor de pasajeros es la capa de presentación, y la terminal final es la capa de aplicación.

📖 Resumen

  • El modelo OSI fue creado por la ISO para estandarizar redes.
  • Son 7 capas interdependientes, de la Física a la Aplicación.
  • Es clave para diagnosticar fallos: ¿dónde se corta el “tren”?
  • Aplicación: HTTP, SMTP, FTP → lo que usa el usuario.
  • Presentación: cifrar, comprimir, traducir.
  • Sesión: abrir/cerrar conexiones.
  • Transporte: entrega fiable (TCP).
  • Red: elegir la mejor ruta (IP).
  • Enlace: evitar colisiones, corregir errores.
  • Física: señales y cables.

Resumen exprés: Física (cables) → Enlace (control) → Red (ruta) → Transporte (orden) → Sesión (abrir/cerrar) → Presentación (traducir/cifrar) → Aplicación (lo que usas).

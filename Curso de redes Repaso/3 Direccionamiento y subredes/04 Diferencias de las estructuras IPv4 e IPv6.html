Bloque A — Conceptos y analogías

A1. En la analogía de los “asientos”, IPv6 aporta principalmente…
Explicación: IPv6 amplía el “aforo” con 128 bits de dirección.
“Más asientos para todos”.
A2. ¿Qué tamaño usa IPv4 para el direccionamiento?
Explicación: 2³² ≈ 4,3 mil millones de direcciones.
Cuatro octetos.
A3. ¿Cuál fue un factor clave para el agotamiento de IPv4?
Explicación: más dispositivos → más direcciones necesarias.
Móviles, IoT, etc.
A4. ¿Para qué se usó NAT en IPv4?
Explicación: NAT permitió “multiplicar” hosts tras pocas IP públicas.
Traducción de direcciones.
A5. ¿Qué tamaño usa IPv6 para el direccionamiento?
Explicación: espacio masivo de direcciones globales.
Ocho bloques hex.

Puntos y decimal.
B7. Formato típico de IPv6:

Resumen rápido

IPv4
  • 32 bits (~4,3×10^9 direcciones)
  • NAT para paliar escasez
  • Clases A/B/C (D multicast, E experimental)
IPv6
  • 128 bits (~3,4×10^38 direcciones)
  • Unicast, multicast, anycast
  • Autoconfiguración y mejor enrutamiento
  • IPsec integrado
Notación
  • IPv4: decimal con puntos
  • IPv6: hexadecimal con “:”
  • :: comprime ceros consecutivos
